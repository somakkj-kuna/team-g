# 관측자료 품질관리(QC) 웹 플랫폼 — 개발 내역 보고

> 작성일: 2026-06-30 · 대상: `qc_webapp` (Flask + 빌드리스 SPA) · 정적 버전 v=25 · 서비스 포트 8002(외부 relay 8501)

---

## 1. 목적

해양 관측자료의 **수집 적시성**과 **품질관리(QC) 결과**를 한 화면에서 통합 점검하고, 분석·보고·다운로드까지 연결하는 운영 지원 플랫폼을 구축한다.

- **3개 기관 통합 모니터링**: 국립해양조사원(KHOA)·기상청(KMA)·국립수산과학원(NIFS)의 관측소 다변수 자료를 하나의 대시보드로 통합.
- **사전 QC 결과 표출**: 웹앱이 QC를 직접 수행하지 않고, 이미 산출된 `flag_final`(품질 플래그)을 그대로 시각화하여 **보존 자료와 제거 자료를 구분**해 보여준다.
- **수집 상태 가시화**: 결측 자료가 "재수집 진행 중"인지 "수집 종료"인지 구분해 운영자가 자료 공백의 성격을 즉시 판단.
- **분석·보고 자동화**: LLM 기반 QC 품질 분석과 한글(HWPX) 보고서 자동 생성으로 보고 작성 부담 경감.
- **자료 활용 지원**: 기관·관측소·변수·기간·품질등급 조건으로 QC 자료를 CSV로 안전하게 내려받기.

## 2. 기능 구현

### 2.1 데이터 기반
- **데이터 모델**: `sample_data`의 사전 QC flag CSV(`time, agency, station_id, lat, lon, var_id, value, depth_m, flag_final, …`) 표출.
- **품질 플래그(`flag_final`)**: `1=양호(good)`, `2=의심(suspect)`, `3=불량(bad)`, `4=보간(interp)`, `9=결측(missing)` → 보존=1·2(·4), 제거=3·9.
- **대상**: KHOA `DT_0001`(인천, 11변수) · KMA `22003`(10변수) · NIFS `bbbi5`(부안 변산, 표/중/저층수온 3변수). 2025년·시간별 자료. 기간(1개월/1년/전체)은 **데이터 최신시점 기준**.

### 2.2 대시보드 / 드릴다운
- **수집 현황 매트릭스**: 요약 카드 + 카테고리별(관측/수치모델/위성) 소스 매트릭스. 관측만 실데이터, 나머지는 "준비중".
- **3단 드릴다운**: 기관 카드 → **변수별 QC 현황**(변수 박스에 양호/의심/불량/보간/결측 통계) → **QC 시계열**.
- **전역 타이핑 검색**(대시보드 상단): 3기관 관측소·수집변수를 인덱싱해 입력 즉시 필터, ↑↓·Enter·클릭으로 해당 시계열로 점프.

### 2.3 QC 시계열 화면
- **flag 구분 표출**: 보존(라인)·의심(주황 ◆)·불량(빨강 ✕)·보간(파랑 ●)·결측(미표시) + 범례 + 통계 카드(전체/보존/제거/제거율).
- **차트 인터랙션**: 마우스 **휠=확대·축소**, **좌드래그=이동**, ＋/－/⤢(전체) 버튼.
- **변수 탭·관측소 탭**으로 동일 화면 내 전환.
- **결측 수집상태 안내**: 결측을 **최신 7일 이내=`자료 수집 진행중`**, **그 이전=`자료 수집 안 함`**으로 시계열 하단에 명시.

### 2.4 AI 분석 / 보고서 (우측 패널, 탭 분리)
- **🤖 LLM 분석 탭**: 자연어 질문 → QC 통계 기반 분석. 대표 질문 칩 제공.
  - **"자료 품질 평가" 칩**: 즉시 분석 대신 **"어떤 관측소와 변수를 분석할까요?" 선택 카드**를 띄워 기관·관측소·변수를 고른 뒤 분석.
  - 분석 결과 아래 **`📄 한글(HWPX)로 저장`** 버튼(해당 분석을 재분석 없이 즉시 보고서화).
- **📄 품질 결과 보고서 탭**: 기관·관측소·**변수(체크박스 다중선택)**·기간 선택 → 보고서 생성.
  - **다중 변수 → 하나의 통합 HWPX**: 개요 + 변수별 QC 통계 요약표 + 변수별 상세(통계표·이상치표·LLM 분석·QC 차트).
  - 생성 중 **회전 스피너 + "보고서 생성 중…"** 진행 표시, 중복요청 방지(`reportBusy`).

### 2.5 QC 자료 다운로드 모달
- **범위 선택**: **기관별 탭** + 관측소 체크박스(다중).
- **변수 선택**: **클릭하면 펼쳐지는 스크롤 체크박스 드롭다운**(전체/개별 다중선택).
- **기간**: 시작일~종료일(비우면 전체).
- **품질등급 필터**: `flag_final ≤ 선택값`("몇 이하만" — 양호만 / 양호+의심 / 불량까지 / 보간까지 / 결측 포함).
- 결과를 **CSV(Excel 한글 BOM)**로 내려받기.

### 2.6 백엔드 / 보안
- **API**: `/api/sources`·`/api/stations`·`/api/variables`·`/api/qc`·`/api/catalog`·`/api/download`·`/api/chat`·`/api/report`.
  - `/api/report`는 단일 변수 또는 **다중 변수(`vars`+`analyses`)**를 받아 통합 1파일 생성, 응답에 `n_vars`.
- **다운로드 보안**: `targets`를 카탈로그 화이트리스트(`valid_targets()`)로만 허용 → 경로/와일드카드(`*:*`)·절대경로·`..` 주입 차단. 사용자 입력은 따옴표까지 이스케이프.
- **견고화**: 시계열 fetch 경쟁 가드(`drawSeq`), Plotly 자원 정리(`purgeSeriesChart`), 에러 시 통계·범례·결측노트 초기화, 보고서 생성 컨텍스트 스냅샷.

> 위 기능은 적대적 멀티에이전트 코드리뷰를 거쳐 보안(다운로드 주입)·경쟁조건·stale 상태 결함을 수정한 뒤 반영됨.

## 3. 확인 가능한 사항 (검증 방법)

### 3.1 서버 상태
```bash
curl -s http://127.0.0.1:8002/api/health        # {"ok":true,"time":...}
```

### 3.2 API 동작
```bash
# 수집 현황 / 관측소 / 변수
curl -s "http://127.0.0.1:8002/api/sources"
curl -s "http://127.0.0.1:8002/api/stations?agency=khoa"
curl -s "http://127.0.0.1:8002/api/variables?agency=khoa&station=DT_0001"

# QC 시계열(+flag 카운트)
curl -s "http://127.0.0.1:8002/api/qc?agency=khoa&station=DT_0001&var=temp&period=all"

# 다중 변수 다운로드(헤더 X-Row-Count 로 행수 확인)
curl -s -D - "http://127.0.0.1:8002/api/download?targets=khoa:DT_0001&vars=temp,sal&maxflag=9" -o out.csv | grep X-Row-Count

# 보안: 와일드카드/경로주입 차단(400 유효 관측소 없음)
curl -s "http://127.0.0.1:8002/api/download?targets=*:*&vars=all"
```

### 3.3 보고서 생성(통합 1파일)
```bash
curl -s -X POST http://127.0.0.1:8002/api/report -H "Content-Type: application/json" \
  -d '{"agency":"khoa","station":"DT_0001","period":"all","vars":["temp","sal"],
       "analyses":{"temp":"# 개요\n- 더미","sal":"# 개요\n- 더미"}}'
# → {"ok":true,"n_vars":2,"filename":"인천_전체기간_QC보고서_2변수_…hwpx","url":...}
```
- 생성 HWPX는 zip 무결성·섹션(개요/변수별 요약표/변수별 상세/QC 차트) 포함 여부로 확인 가능.

### 3.4 UI 동작 확인 절차 (브라우저)
1. 대시보드에서 **전역 검색** → 관측소/변수 입력 시 즉시 필터·점프.
2. 기관 카드 → 변수 박스 → 변수 클릭 → **시계열**(휠 확대/축소, 드래그 이동, flag 색 구분, 결측 수집상태 문구).
3. 우측 **LLM 분석** 탭: "자료 품질 평가" 칩 → 관측소·변수 선택 카드 → 분석 → 분석 아래 HWPX 저장.
4. 우측 **품질 결과 보고서** 탭: 변수 체크박스 다중선택 → 생성 중 **회전 스피너** → 완료 시 단일 통합 파일 다운로드 링크.
5. 상단 **데이터 다운로드** 모달: 기관 탭·관측소 체크·변수 드롭다운·기간·품질등급 → CSV 다운로드.

## 4. 표출 방안

### 4.1 접근 경로
- **내부**: `http://<host>:8002/` (Flask 직접 서빙).
- **외부**: relay를 통해 **포트 8501 → 8002**. (동일 서버의 다른 서비스: 8000 NOSC 수온, 8001 연안침수와 분리 운영.)

### 4.2 화면 구성(표출 레이아웃)
- **좌측(메인)**: 수집 현황 대시보드 → 드릴다운(변수별 QC 통계 → QC 시계열).
- **우측(패널)**: 탭으로 **LLM 분석** / **품질 결과 보고서** 전환.
- **상단**: 전역 검색 + 데이터 다운로드 버튼.
- 라이트 테마·네이비 헤더·Pretendard 글꼴, 로컬 Plotly(외부 의존 없음).

### 4.3 산출물(표출 형식)
| 형식 | 내용 | 경로/방법 |
|---|---|---|
| 웹 시각화 | QC 시계열(Plotly), 통계 카드, 수집 매트릭스 | 브라우저 대시보드 |
| 한글 보고서 | 변수별 통계·이상치·LLM 분석·차트 통합 HWPX | 보고서 탭 / `POST /api/report` → `reports/` |
| CSV 자료 | 품질등급·기간 필터 QC 자료(BOM) | 다운로드 모달 / `GET /api/download` |

### 4.4 운영 유의
- 정적 자산은 캐시 무력화를 위해 `index.html`의 `?v=N`을 수정마다 증가(현재 v=25).
- 파이썬 코드 변경 시 서버 재기동 필요(`fuser -k 8002/tcp` + `setsid`).
- `sample_data`는 비공개(공개 repo 커밋 금지).
